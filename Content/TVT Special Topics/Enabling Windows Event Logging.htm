34.5 Enabling Windows Event Logging

34.5.1 Prerequisites

Event logging is enabled by editing a configuration file. This file resides in a directory that is protected (by Microsoft Windows) against modification by ordinary users. Editing the configuration file, therefore, requires local administrator rights for the computer on which logging is to be enabled.

34.5.2 Step 1: Open the Configuration File

Logging is enabled by editing the configuration file “override.tvtdef”. Check whether an “override.tvtdef” file exist in the TVT program directory (usually C:\Program Files\SHP Text Verification Tool 12.0.0). If not, please see See Special Topics "Get the Configuration File" on page 1 User Name and copy the "override.tvtdef" file from the "Windows Event Logging" folder to the TVT program directory.

Please make sure that the file is edited only with a plain text editor! Do not use any other program, such as a word processor (e.g. Microsoft Word). The Windows “Notepad” application is acceptable; the Windows “WordPad” application is not.

As indicated in the prerequisites, it is necessary to start your editor with administrative rights; otherwise Windows will prevent you from saving your modifications.

34.5.3 Step 2: Set the Logging Policy

Find the following two lines in the configuration file:

<PROPERTY ID="WINEVENTLOG_PROGRAM" VALUE="0"/>

<PROPERTY ID="WINEVENTLOG_PROJECT" VALUE="0"/>

The logging policy is controlled by the two highlighted values:

  • “WINEVENTLOG_PROGRAM” : Setting the value in this line to “1” causes TVT to make one log entry every time the TVT application is started, and another log entry every time the TVT application is exited. The default value “0” disables logging of startup and exit events.
  • “WINEVENTLOG_PROJECT” : This value can be set to either “1” or “2”. The default value “0” disables logging of project events.
  • VALUE="1" : This setting causes TVT to make one log entry every time a new project is started in TVT, and another log entry every time a project is closed.
  • VALUE="2" : In addition to starting and closing of a project, this setting also makes log entries for the following events: saving a project, loading a project, comparing a project, writing a project report file, exporting the annotated original or copy files, and project reset.

Make sure that you change only the highlighted digit between the two quotes in VALUE="0". Do not remove or alter the quotes around the digit. In addition, you should not change any other part of the line or any other line of the configuration file.

34.5.4 Step 3: Save the Configuration File

To complete the “Event Logging” process you must save the altered configuration file. If you have removed any write protections from the configuration file (e.g. to circumvent the need for starting your editor with administrative rights), make sure that you restore those protections to their original state.
