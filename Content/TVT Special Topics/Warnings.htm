20 Warnings

This feature is initialized via the Project settings, but almost identical for all company independent settings. (“Flip” warnings are exclusive to “Complex Languages” settings.) Within a project, it can be controlled by using the “Warning Settings” dialog shown above. It is accessible under the “Project” > “Warning Settings…” menu.

Warnings are defined as mapped deviation pairs. They occur when non-identical words get mapped in the comparison process. This can only happen if both words have the same length and the differing characters are in one group of the so-called UnicodeX table or have been declared compatible by the Unicode Standard. UnicodeX Warnings are divided into different categories, which are referenced by groups. Each group contains a set of equivalent characters.

Mapping slightly deviating words has several advantages:

  • The comparison process creates fewer text blocks and deviations
  • If it is not necessary to analyze a transition or category, it can be toggled off completely instead of dealing with each map warning.

Each category (other than ligature) may have one or multiple groups that define sets of interchangeable characters. E.g., if “u” and “a” were defined in the same group, “run” could be mapped to “ran” – creating a warning.

If characters cannot be mapped by the above categories, TVT uses the Unicode Standard in an attempt to determine a relationship dynamically. The following categories are to be distinguished:

Upper to Lower Case Transition (41.3.1 "Upper to Lower Case Transition Category" on page 1)
Lower to Upper Case Transition (41.3.2 "Lower to Upper Case Transition Category" on page 1)
Unicode Compatibility (33.1 "Unicode Compatibility Category" on page 1)
Illegal Character (27.2 "Illegal Character Category" on page 1)
Private Use (9.8 "Private Use Category" on page 1)
Soft Hyphen (18.9 "Soft Hyphen Category" on page 1)

Minor and major spacing warnings are different from the other map warnings. They do not represent character changes but missing or inserted spaces in one of the documents (see 24.1 "Spacing Warnings" on page 1”).

No-Break Warnings occur, when a non-breaking space, preventing an automatic line break at its position, is included in a document, but the line break has taken place on the document it is compared to.

A special kind of Warnings are the Elongation Warnings. They appear, when elongation insertions, known as Kashida elongations, exist between Arabic characters.
