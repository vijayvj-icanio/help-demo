18.2 Barcode Decoding

Following barcode types can be detected and decoded by TVT Barcode:

  • EAN 13
    EAN 13 barcodes whose first digit is zero are indistinguishable (to a barcode reader) from UPC A barcodes. They use the same barcode symbols, and are only distinguished by the human-readable part. If both EAN 13 and UPC A  are selected for detection in the User Settings, TVT Barcode will recognize such codes as EAN-13. If EAN-13 is selected and UPC-A is not selected for detection, TVT Barcode will not detect any EAN-13 barcode with a leading zero.
  • UPC A
  • EAN 8
  • UPC E
  • Code3of9
  • Code3of9extended
    Extended encoding can encode a greater range of characters by repurposing some of Code 39’s symbols as shift operators. These are combined with another character to represent characters unavailable in the non-extended encoding. Not all combinations of shift and other characters are permitted. TVT will display the character “ÿ” as a placeholder wherever such an invalid sequence occurs.
    It is a known issue that barcodes in extended encoding that contain the ASCII NUL character (code point 0) are either recognized truncated, or not recognized at all.
  • Code 128 / UCCEAN 128
    It is a known issue that Code 128 barcodes that contain the ASCII NUL character (code point 0) are either recognized truncated, or not recognized at all.
    Using the FNC4 symbol as a shift operator, which modifies following characters in order to encode the non-ASCII code points 128 to 255, is not supported.
  • Interleaved2of5
  • CodaBar
    The first and last symbols in a CodaBar barcode are start and stop symbols, of which there are four. They are often referred to by the names A, B, C, and D. TVT will treat and display them as part of the data, as lowercase characters. For example, a barcode with start symbol "A", stop symbol "C", and data “1234” is displayed “a1234c”.
  • Code93.
  • EANEXT 5
  • EANEXT 2
  • MSI
    Only barcodes with a single "modulo 10" check digit are supported. MSI barcodes using other checksum schemes will not be recognized. The check digit will not be treated as part of the barcode data.
  • Code 11
    Only barcodes with a single "C" check digit are supported. Code 11 barcodes using other checksum schemes will not be recognized. The check digit will not be treated as part of the barcode data.
  • Standard2of5
  • PatchCode
    When a Patch Code is read, its "data" is set to a four-digit sequence of zeros and ones, with zeros representing thin bars, and ones representing thick bars.
  • Postnet
  • Planet
  • Australia Post
  • Royal Mail 4SCC
  • US Postal Service 4BC
  • GS1 DataBar (formerly RSS-14), GS1 DataBar Limited, and GS1 DataBar Stacked
  • Some barcodes that have their "linkage flag" set are not supported. The linkage flag can indicate the presence of a 2D component in a GS1 Composite barcode.
  • QR Code
    Barcodes with Extended Channel Interpretation (ECI) are not supported.
  • DataMatrix
    Barcodes with Extended Channel Interpretation (ECI) are not supported.
