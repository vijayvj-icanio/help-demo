35 Style Check

As TVT is designed to tolerate document reformatting, only character styles are of interest for style checking. Paragraph and other style information, such as multiple columns on a page, are not processed. In addition, styles that are not imported (see 24.5.6.6 "Text Style Attributes" on page 1 and 16.6.1 "Graphical Representation" on page 1) cannot be checked.

To start the style check, select the Style Check tab from the Project Window toolbar. Once you have selected it, all of the style transitions between the documents are displayed in the Style Transitions list below.


Style Transitions List Tab

In this list, it can be easily recognized if a change in font or font attribute (e.g. bold or italic) has been carried forward. All style transitions in the list are initially displayed as unchecked. Any transitions that are not correct should be checked/marked, either manually, or by using the Automated Style Check explained below. Checked/marked transitions are considered as errors and receive an automated comment accordingly. The “Amount” column lists the number of mapped words that create the indicated style transition. The list is sorted by quantity, since transitions with a higher quantity are usually intended, while transitions with only a few instances are not.

When an instance of a transition is selected, TVT highlights all identical styles in the same document for ease of location (see figure ‘a’). In this case “Times New Roman; 12.0 pt” in the original has been converted to “TimesNewRomanPSMT; 12.0 pt; underlined”, but also to “TimesNewRomanPSMT; 12.0 pt” etc.


Figure ‘a’

Automated style check may be performed by clicking the Automated Style Check button at the bottom of the Style Transitions list. There are four style check modes, as well as different Style Conformity options, which can be then selected from the pop-up dialog:

The Automated Style Check has the advantage that comments, in accordance to the selected style check mode and/or Style Conformity option, are automatically added to the corresponding style transitions. These automatically created comments cannot be customized. However, they may be manually overwritten for each style transition. Any manually added comments are then kept and not overwritten by the Automated Style Check. To clear all comment fields, after a manually or automatically performed Style Check, use the “Uncheck All” button at the bottom right of the Style Transitions list.

When a mode is selected, the transitions that are not valid for that mode are highlighted and checked in the Style Transitions list (see Figure ‘y’).


Figure ‘y’

In order to be considered as performed, the Style Check should be confirmed with “OK” at the bottom left of the Style Transitions list.
