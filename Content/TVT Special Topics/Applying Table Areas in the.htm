20.1.1 Applying Table Areas in the Compared State

It is also possible to add Table Areas in the compared state and to apply the selected reading order once.

Please note that placing Table Areas before comparison remains the recommended usage of the Table Area Mode. Table Areas have a greater chance of changing the comparison results for the better if they are placed before comparison. The possibility of applying Table Areas in the compared state is a convenience in case you have forgotten to place them before comparison, and have not noticed this until after you have made a significant progress working on your project.

Applying Table Areas in the compared state has a number of caveats:

  1. It is not compatible with existing Style Check results. If you try to apply a Table Area after perusing the Style Check Project Window, you will be prompted with a dialog asking you whether you would like to discard the results of your Style Check. Your choice is to proceed (which discards your Style Check and applies the Table Area) or reset (which keeps Style Check results, but does not apply your Table Area.)
  2. It does not change the outcomes of a Critical Word check: The Critical Word functionality always marks those words that were considered critical at the time when the Compare button was pressed. Applying a Table Area in the compared state does neither mark words as critical that match your Critical Word list only after the Table Area has reordered the text, nor does it remove Critical Word markings from text that matched your Critical Word list before reordering, but not after.
  3. If you have done work on Deviation List entries that include words under the Table Area, then this work (i.e. comments and marking) will be lost, and the Deviation List entries will be reset to the unprocessed state.
  4. Applying Table Areas in the compared state is not available when the Clone feature is in use (i.e. in the Cartons settings.)
  5. Applying a Table Area in the compared state attempts to map text under the Table Area based on how this text was mapped before applying the Table Area. This means that a) if no text was mapped before applying, none will be mapped after applying; and b) if the mapping was wrong before applying, then the mapping may still be wrong after applying.

You can avoid all of these problems by placing your Table Areas before pressing the Compare button, or by resetting your project and recomparing, if you find text that needs Table Areas after the comparison.
