36.12 Ligature Category (Multi-Character UnicodeX Warnings)

The Unicode standard contains a number of code points for digraphs and ligatures. These are single Unicode values whose glyphs represent or appear as two or more letters. TVT can recognize substitutions of such characters with their decomposed form, i.e. the character sequence that composes or appears to compose the code point. For example, TVT will recognize a warning, not a deviation, when “coffee” (with two “f” characters) is compared with “coffee” (with the “ff” ligature U+FB00). The list of such permitted substitutions is defined in the UnicodeX table.

Ligature warnings will not be recognized under the following circumstances:

  • when compared against hyphenations: “coffee” (with “ff” ligature) vs. “cof-
    fee” (with a hyphenation in the “ff” sequence)
  • when compared against style changes: “coffee” (with “ff” ligature) vs. “coffee” (with a style change in the “ff” sequence)

 

The Compare Selections Area displays a gray ellipsis character “…” in order align and compare words that have Ligature warnings. This ellipsis should not be misconstrued as an ellipsis character in the word. The apparent ellipsis disappears when shifting the words changes the character sequence.
