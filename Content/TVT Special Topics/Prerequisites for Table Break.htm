36.15.7 Prerequisites for Table Break Detection

As stated in 18.7 "Table Breaks" on page 1, the Table Break feature is built on top of the Table Area facility. This means that Table Breaks can be detected only in tables that are correctly covered by Table Areas. There are four conditions that need to be fulfilled for the Table Breaks feature to work:

  1. Table Areas must cover the tables on both sides, in the original and copy documents. It is not sufficient to have a table area on one side. This is notably different from using Table Areas for reading order correction, where it is usually sufficient to correct the reading order on one side.

û

ü

 

  1. Table Breaks can be detected only when they occur within one Table Area. If a table is covered by multiple Table Areas, then Table Breaks that occur between one Table Area and another will not be detected.

û

 

  1. Table Areas must be placed so that columns are detected correctly (as indicated by the white columns within the green Table Areas). This can be difficult when tables have horizontally merged cells that span multiple columns.

û

ü

 

  1. Only mapped text is considered for Table Break detection. Unmapped text cannot be detected as a Table Break but can be detected as some other type of Deviation.
