16.6.1 Graphical Representation

RTF documents are represented by an internal engine. This engine supports the following Word features:

16.6.1.1 Character Base

  • Font face (if the font is installed on the system)
  • Font size
  • Italic attribute
  • Bold attribute
  • Underline attributes (All underline variants are represented as regular underlines)
  • Strike-through attribute
  • Double-strike-through (represented as regular strike-through)
  • Uppercase attribute (supported by character conversion)
  • Superscript
  • Subscript
  • Tabulators
  • All capitals (supported by character conversion)
  • Text color (White text on white background is imported, but will not be visible. Use monochrome mode to see such text.)
  • Vertical shift

16.6.1.2 Paragraph Base

  • Left indent
  • First line left indent
  • Page breaks
  • Left Aligned
  • Centered
  • Right Aligned
  • Margin above
  • Margin under
  • Line spacing
  • Bullets/Numbering

16.6.1.3 Others

  • Table support is simplified and primarily designed to show the content in the correct position.
  • The following table styles are used:
  • Table width
  • Cell width
  • Left indent
  • Right indent
  • Simple cell borders (not all line styles are supported)
  • Inner cell margins
  • Footnotes are placed under a horizontal line at the bottom of the page:
  • In multi-page tables, footnotes are sometimes moved to the next page
  • Footnote identifiers are always in decimal notation, regardless of the counting mechanism used in MS Word
  • Endnotes are treated like footnotes and placed at the bottom of the page, not at the end of the document
  • Text from text boxes is placed at the anchor location, surrounded by a light gray border. (This border is omitted when printing.) Please note that in Microsoft Word, the anchor location may be different from the location of the text box.
  • Drawing shapes are treated like text boxes; i.e. the text they contain, if any, is put in a light gray box at their anchor location. Their graphical appearance is not reproduced. This includes their background color, which will not be imported, while text color is imported. Note that this causes white text on a colored background to be imported as white text on a white page.
  • Embedded images are supported with the following caveats:
  • Images are shown at their anchor location
  • Borders are not supported
  • Support for image rotation is limited to multiples of 90Â°
  • Images that are based on metafiles (.wmf and .emf) may look distorted or blurry
  • TVT also attempts to load linked (external) images. If a linked image cannot be loaded, it is replaced with a default error marker

16.6.1.4 Explicitly unsupported Word features

  • Page headers/footers
  • Floating objects
  • Embedded COM objects and stored documents (e.g. Excel Sheets, PDF and MS Word documents).
