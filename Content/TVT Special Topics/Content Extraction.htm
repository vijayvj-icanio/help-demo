24.5.6 Content Extraction

The primary function of PDF files is to display identical output on different operating systems. Text extraction is deemed secondary and, therefore, not always supported. However, text extraction is necessary for TVT, for indexing in a document management system and other services.

There are some issues relating to text extraction from PDF documents. The most important ones are described below:

24.5.6.1 Content Transformation

Some PDF creators can be configured to produce curves (Vector Files) instead of text . With Vector Files, it is not possible to select text in Adobe Reader. Similarly, if the text has been transformed into an image (Pixels), TVT will not be able to extract it.

24.5.6.2 Encoding Transformation

PDF textual content is encoded for use with a specific font. During text extraction, this content is converted into Unicode with the help of “toUnicode” tables stored in the document. Sometimes, these tables contain invalid values, leaving the extracted text unreadable. In TVT, you can check this by selecting a word and viewing it in the Details Area.

In the Details Area, the textual content of the word is shown, not its graphical representation. If the word content is different from its appearance in the PDF document, your document is incorrectly encoded.

Alternatively, you can copy a text segment from your PDF Viewer application (e.g. Adobe Reader) to the clipboard and paste it into Microsoft Notepad. If the text is unreadable there, it is also unreadable for TVT.

There are issues known to cause this problem that are related to the software as well as the fonts used to create PDF documents.

Use of certain PDF creation tools

Besides the Adobe Distiller, there are many printer driver emulators, plug-ins and applications that create PDF documents. Some of them may not be able to deliver correct “toUnicode” tables under certain (or all) conditions. If you are experiencing problems using a certain PDF creator, try using another one. Generally, the Adobe Distiller or another printer driver emulator can be used as a fallback.

The Adobe Distiller does not provide all PDF functionality (e.g. interactive features like hyperlinks are not exported), though graphical and content representation stay intact.

24.5.6.3 Positioning Transformation

PDF content is divided into text strings with positioning information. During the text extraction, this positioning information is used to decide, if the word at the end of one string is connected to the beginning of the next one. Under rare circumstances, this procedure fails and delivers individual words as one word or vice versa. There have also been instances, where the positioning information did not match the text placement exactly. You can check if a word was imported correctly by selecting it in TVT. If a word surrounding, only a part of it, nothing or space beside the word becomes highlighted, you are experiencing this problem. In this case, you should provide us with example documents.

24.5.6.5 Security and Permissions

PDF documents feature a user permission management that has to be enforced by TVT. If a restriction is activated which is relevant for TVT, the appropriate (user or owner) password will be requested by TVT. If the password is not accepted for the third time, the document will not be opened in TVT and a blank page will be shown instead.

The following restrictions are relevant for TVT:

  • The document must not have a user password or the correct user password must be provided
  • The document must not restrict text extraction and annotation adding or the correct owner password must be provided.

User passwords restrict the access to the document itself. Owner passwords restrict the modification of security attributes and protected areas of PDF documents.

24.5.6.6 Text Style Attributes

Font Name

The font name is extracted reliably. However, PDF creators usually mangle font names in several ways. Font names therefore do not necessarily appear exactly as they did in the application from which the PDF file was exported. The most common changes are:

  • All spaces are removed from the font name
  • A suffix is appended (e.g. “MT”, “PSMT”), often for disambiguation between different styles of a font family (e.g. “Bold”, “Italic”, “BoldItalic”)

Font Size

This value is extracted reliably. For PDFs exported from Microsoft Word it may vary slightly (e.g. 0.1 pt) from the value assigned in Word.

Font Color

PDF documents feature a variety of color spaces. TVT translates text color into RGB values, where 0 represents black and 255 the maximal color value for each component. This includes text whose color is in a CMYK color space. Converting color values from CMYK to RGB is not a trivial operation and usually not possible without inaccuracies. Please take this into account when inspecting RGB color values in TVT.

Bold Attribute

Fonts that are embedded in PDF files usually come with a font descriptor, which may optionally contain information about the font's weight, indicating whether it is bold or not. In the case that a PDF creator did not provide this optional attribute, hints about a font's weight can sometimes be found in its (mangled) font name. For purposes of the simplified style check, a font is regarded as bold if its name contains one of the following case-sensitive tags:

  • Black, black, Blk, Bold, bold, Bd, Heavy, heavy, Hv, Fett, fett, Bol

Generally, tags starting with an uppercase letter are matched only at the end of a font name, or if followed by another uppercase letter.

Italic / Skew Attribute

There are three different ways to determine if text in a PDF document is italic:

  1. The font descriptor should contain information whether the font is italic or not.
  2. For purposes of the simplified style check, a font is regarded as italic if its (mangled) name contains one of the following tags:
  • Inclined, inclined, Ic, Oblique, oblique, Obl, Italic, italic, It, Kursiv, kursiv, Ks, Sloped, sloped, Slanted, slanted, Sl, Ita.
  1. If the bounding box that encompasses a character is sufficiently “skewed” or “slanted” rather than upright, then that character's style is assumed to be italic. Slanted characters are usually the result of PDF creators synthezising a pseudo-italic version for a font family that does not come with an italic variant.

As a result, the information about italic style may be present in the font name, the skew value and/or the italic flag or may not be present at all depending on the fonts used.

Underlining / Strike-through Attribute

These are not font attributes in PDF documents, but vector information that needs to be brought into relation with the text. TVT uses the position of the line and the word, as well as the color.

It is dependent on assumptions regarding the correlation of the underline/strike-through position, the character offset point and the character boxes. These assumptions may be wrong for certain fonts. As a consequence, other lines may be erroneously detected as underlines (e.g. the bottom line of a table cell).

Only words that are completely underlined or struck through will get an “Underline” or “Strike-through” tag. Words that are only partially struck through or underlined will be marked as “Partially Underlined” or “Partially Struck”. Words that have significantly broken or dotted lines will also be recognized as partial.

As a result, the reliability of underline and strike-through information depends on the workflow (font usage and page layout included).

Vertical Shift / Sub- / Superscript Attributes

There is no such font attribute as “Subscript” or “Superscript” in PDF documents, so these attributes do not get extracted directly. But the positions of characters that are assumed to be in the same line can be related to each other in order to determine a vertical shift. (The simplified style check knows how to compare sub-/superscript attributes from non-PDF files to the resulting vertical shifts in PDFs.)

This vertical shift detection is a heuristic; in rare cases it may yield incorrect results.

Summary:

Some style attributes, such as bold or italic, are not always reliably provided by the PDF format. But sometimes the font name can indicate the missing information by proxy.

Other attributes, such as underlining and strike-through, are reflected only in graphical (non-textual) data. Sub- and superscript are indicated only by font size and vertical text position. In these cases, determining the underlying style attributes requires heuristics.

All this makes it impossible to gather style attributes from text in PDF documents in the same way as for other document types.

However, these issues do not affect text comparison, only style comparison – which is discussed in 35 "Style Check" on page 1.

If style comparison or conformity checks are part of your application requirements:

  • Style import tests based on realistic documents should be included in your test procedure (e.g. within the product validation).
  • PDF documents will have to be created in a defined workflow that is known to create coherent results.

24.5.6.7 Interactive Content

PDF files can optionally contain various kinds of interactive or other non-printable content. Such content is not evaluated by TVT for comparison purposes. Examples of such content are:

  • Embedded movies and sound files.
  • Embedded Adobe Flash or ActionScript programs. This includes “PDF Portfolios” which use an embedded program to present an interactive menu for selecting subdocuments to the user.
  • PDF Forms: Forms are usually normal (i.e. printing) content that can be compared, but pre-filled, suggested, or calculated values inside of form fields might be realized as interactive content.
  • XFA Form PDFs, usually generated by Adobe LifeCycle, cannot have their content extracted or compared.
