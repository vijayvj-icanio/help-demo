18.3 Server Installations Security

Installations of Text Verification Tool on enterprise servers give a lot of opportunities for providing larger user groups with the application, while keeping efforts for maintenance low. Solutions, such as Citrix XenApp and others, are used to accomplish this requirement.

Nevertheless, these shared environments require an additional thought when it comes to security. This is not a TVT specific, but a good practice for multi-user environments. In a Standard Windows Server environment, users could:

  • see the names of other users
  • create own content in C:\ root folder
  • have read access to program files and configurations, which might not be desirable.

It is a good idea to block access for users via a Windows group policy. This way the users cannot access the C drive via the explorer during file selection.


Example on how to prevent access via local group policy

TVT stores some of its configuration files in the program directory, therefore, just blocking the C drive would not allow the user to access this configuration files anymore.

To allow users access to the TVT configuration files and examples, we would recommend the following:

  1. Create a new drive (e.g. D) for use with TVT. The users should only have read access to the drive and all of its folders.
  2. Create the folder TVT and create a subfolder TVT12.0.
  3. Copy the folders “Additional Input” and “Examples” into this folder.
  4. Check whether an "override.tvtdef" file exist in the program directory. If not, please see See Special Topics "Get the Configuration File" on page 1 User Name
  5. Open the "override.tvtdef" file with a plain text editor and locate the DIRECTORIES section.
  6. In the line with FOLDER_ADDIN, change the VALUE to the new additional input path.
    <PROPERTY ID="FOLDER_ADDIN" VALUE="D:\TVT\TVT12.0\Additional Input"/>
  7. Now you can enable the policy to prevent access for the C drive.

To allow quicker access to the Examples files and further folders, you can add these to the Open File Dialog in TVT as follows:

  1. Open the "override.tvtdef" file as described above and locate the NEWPROJECTWIZARD section.
  2. In the line with QUICK_PATH, change the VALUE in accordance with the location of the Examples folder (see point 3 in the listing above).
  3. To add further folders, separate them by semicolons.
  4. The result in the Open File Dialog will look like that for
    VALUE="%PROGRAM%\Examples;C:\Test".
