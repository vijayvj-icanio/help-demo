36.14 Understanding the Table Area Mode

When transferring information contained within tables from one document to another and from one format to another, the order flow of the text can be altered even though the visual appearance remains the same. For instance, when the text from the Original Table ‘a’ is reformatted in the Copy Table ‘b’ the resulting Inspection ‘c’ appears to have some deviations.

These deviations are caused not by differences in the text but by a difference in the way the text flows from word to word in the two tables. This then requires the user to manually map the text that is not mapped. In this instance the mapping is relatively straightforward, but some tables containing many cells can take longer to map. When the text is mapped in the document as in Figure ‘d’ below, the order flow differences that have created the non-mapping of text become apparent.


Figure ‘d’

The arrows show the order of flow in each table. In the document on top (above) the word order is

1 → 189 → 110 → 44 → 2 → 167→ 78 → 70→ 3→ 177→ 88→ 54,

whereas in the document on the bottom (above) on the right the word order is

1 → 2 → 3 → 189 → 167 → 177 → 110 → 78 → 88 → 44 → 70 → 54

Because the words are in cells and isolated, TVT cannot map the text in the usual way. In this case there is unmapped text for the user to deal with.

By using the Table mode, it is possible to reorder the text flow so that the software maps the text without manual mapping. In the following example the table mode has been activated and the “Order by line” option has been selected.

Preparation for Inspection with Table Mode ‘e’

This means that the text flow will be dictated by the TVT and will follow the line order as displayed, which then matches the line order in the top document as shown in the left hand document above. This enables TVT to map the text without manual mapping as in “Completed Inspection with Table Mode ‘f’”. There are some instances where table mode cannot map the text in the cells, but the table mode allows easier inspection of complex tables.
