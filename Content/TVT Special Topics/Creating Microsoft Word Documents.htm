40.6 Creating Microsoft Word Documents

Usually, Word documents are used as manuscripts while PDF documents contain the artwork. As mentioned in 19.12 "Rich Text Format (RTF) Import" on page 1, Word documents may contain features not recognizable by TVT. Therefore, the key is to keep the manuscript (the original Word document) as simple as possible. In general, use only Word features that will be represented in the PDF document generated from it.

40.6.1 Use page breaks to separate content

RTF documents have a different visualization in Word and in TVT. Therefore, it may not be easy to locate content. You can use page breaks to separate content in Word and in TVT, making it easier to find content.

40.6.2 Use picture references instead of pictures

Pictures in RTF files can only be displayed in a simplified manner. Bitmaps stored in RTF documents use up much more memory than externally stored bitmaps and increase loading time significantly both in Word and in TVT. Use picture references such as “[Image 1a]” in your manuscript and supply your artwork creator with separate images. In the TVT project, the text [Image 1a] will get marked as “No Match” and you can manually verify whether the artwork creator has used the correct picture.

40.6.3 Use only MS Word features that are significant for the PDF creator

There is an almost infinite number of formatting options in Word, but only a subset of important features is supported by TVT. As far as possible, restrict yourself to the format options listed in 16.6.1 "Graphical Representation" on page 1. Please note that there are formatting options that affect the text import. Therefore, do not use Word frames or Word page headers/footers for text you want to compare using TVT.

40.6.4 Font use

Only use fonts that are commonly available. Do not embed fonts into RTF or Word documents. As the layout artist will use other fonts anyway, font changes are unavoidable. Use symbols from the Unicode table rather than symbols from a symbol font. This will reduce the number of character code transformations and, therefore, the chance of an error. The only symbol font fully supported by TVT is the font “Symbol”. Do not use “Wingdings”, “Zapfdingbats” or other symbol fonts.

40.6.5 Use of Greek symbols

You can use the “Symbol” font to enter Greek symbols. Please note that there is a PDF text extraction error for the symbol φ (Greek Phi Symbol, U+03D5) and ϕ (Greek Small Letter Phi, U+03C6), which is based on a mistake in Unicode Standard 3.0. These symbols will be swapped during text extraction, at least when created with the Acrobat 5 Distiller. In the PDF document these symbols are shown correctly.

40.6.6 Disable change tracking

Before importing an RTF document in TVT, uncheck track changes and accept or reject all changes. For DOC documents, all changes are accepted automatically. Of course, your original files remain unmodified.

40.6.7 Keep tables simple

Try to use only table styles that are directly supported by TVT (please see chapter 16.6.1 "Graphical Representation" on page 1). If you have to use table styles that are not directly supported by TVT, the table content may be relocated and table attributes (such as borders, background, etc.) will eventually be invisible.

Word appears to have bugs that occur when editing tables (deleting or merging rows, columns, or whole tables) with enabled change tracking. These can be avoided by temporarily disabling change tracking before editing tables. Saving in the .DOCX format can help to salvage files which contain tables that have been broken by bugs in Word. The .DOCX format sometimes filters out the broken table constructs.

40.6.8 Separate tabular information in cells

Arrange information in tables logically. If you have an array of values, place each value in its own table cell. TVT will parse the table cell by cell and, therefore, retain the correct reading order (please see chapter 36.14 "Understanding the Table Area Mode" on page 1).
