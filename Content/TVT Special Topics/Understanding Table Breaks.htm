40.15.5 Understanding Table Breaks

Table Breaks are detected according to the following principle: the column relation of two words in the same table should be constant. The column relation is based on a simple comparison of the horizontal position of columns containing the words. The column relation of two words in the same table can have one of three possible values:

Cat       Cat   Cat  
  Mouse   Mouse     Mouse  

‘Cat’ left of ‘Mouse’

  

‘Cat’ right of ‘Mouse’

  

same column
  1. The two words are in different columns, and the column containing the first word is to the left of the column containing the second word.
  2. The two words are in different columns, and the column containing the first word is to the right of the column containing the second word.
  3. Both words are in the same column. (Their relative horizontal position within that column does not matter.)

A constant column relation means that the column relation of two words should be the same in the original and copy documents. No Table Breaks will be reported as long as this is the case. If two words have one column relation in the original document, but another in the copy document, this will be detected as a Table Break.

The first example increased the distance between the words; the second example moved them equidistantly. In both cases the column relation remains unchanged “left” on both sides. That neither case is detected as a Table Break is by design; this behavior avoids detecting harmless layout changes as Table Breaks. If the tables had more text, e.g. in a third row, and that text was not moved, then a Table Break between that text and the moved text would be detected.

As mentioned before, Table Breaks are listed in the Order Break category of the Deviation List. They are not visualized in the documents. A Table Break usually results in two Deviation List entries: a “Table Break After” highlights the element before which the break occurs, and a “Table Break Before” highlights the element after which the break occurs.

When the same break causes a column relation change for many words, only the words closest to the break are marked for the Table Break. Consider the example above: The column relation between “cuddly” and “zebra” changes from “same” in the original to “left” in the copy. But the TVT indicates the Table Break between “dog” and “meek,” which is closer to the break location and, therefore, more helpful in identifying the cause of the Table Break.

The “Table Break After” and “Table Break Before” entries do not necessarily point to adjacent words. There might be a number of words between them for which the column relation has not changed. In the following example, moving the “zany zebra” changes its column relation to the “cuddly cat.” But the column relation to the text in the third column remains unchanged. Therefore, the closest words that indicate the Table Break are “cat” and “zany.”
