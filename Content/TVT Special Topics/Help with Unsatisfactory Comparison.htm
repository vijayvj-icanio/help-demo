23 Help with Unsatisfactory Comparison Results

You have compared two documents and only little text was mapped. What went wrong? Take a look at the following headlines and see if one matches your screen output. Please note that these are just exemplary cases – in reality they may be mixed to almost any degree.

23.1 Most of the text in both documents are marked orange, only a little text is not marked or marked yellow

If a word is marked orange after comparison, it does not appear in the other document. So if documents are almost completely marked orange, they have almost no common words. Perhaps they are not written in the same language.

Tip
Please verify that you have loaded the correct documents into the project.

23.2 Most words are marked yellow in both documents; only few words are unmarked or marked orange

Yellow markings indicate words that are present in the other document but cannot be mapped without doubt. Usually this happens when one or both documents contain large duplicate parts.

Tip 1:

  • Close the project
  • Open the “Project Setup” and select your documents
  • Click the “Change” button to open the project setting dialog
  • Select the “Documents” project settings and click OK
  • Finalize creation of your project in the Project Setup by clicking OK
  • Compare the documents again.

Tip 2:
Before the comparison, check the original and copy word count in the status bar. If the copy contains many more words than the original, it may be difficult for TVT to automatically map the correct part. Perhaps the copy contains more than one version of the source document. Perhaps it contains a color and a monochrome version or versions intended for two different dosages of a medication. In that case, follow these instructions:

  • Reset the project
  • Disregard all but one version of the text in the copy by using Exclusion Areas and/or Document Dividers, so that the remaining one can be checked.

Tip 3:
If both documents have about the same number of words, both might still contain more than one version of the same text. You can use document dividers to select only one version in each document. Then, the comparison can be repeated for all other versions in the same manner. For further information, see 34.7 "How to Use Document Dividers" on page 1.

Tip 4:
If there are only a few (orange marked) unmatched words and large yellow areas after comparison, you can use the “Manual Map” function (explained 19.9.1 Manual Map C:\TFS\Docs\FlareDoku\TVT Documentation\Output\Yuliana_Wong\TVT Web New Target\Content\TVT Reference Guide\Manual Map.htm file:///C:/TFS/Docs/FlareDoku/TVT Documentation/Output/Yuliana_Wong/TVT Web New Target/Content/TVT Reference Guide/Manual Map.htm Manual Map 1 1 on page 1) to map segments of the document. You will probably be able to map the document with just a few steps.

23.3 Both documents contain an even mix of yellow, orange and unmarked text

Tip 1:
Please check whether you are comparing package artwork. In such projects, text has a very different layout in the original and the copy, so it is not possible to map large segments. Some text is repeated in the copy (e.g. Brand Names) but appears only once in the original.

In order to compare the documents efficiently, use the “Cartons” project settings, with the manuscript as original and the carton artwork as copy. Use the “Manual Unmap/Map” mode to set up the main associations between manuscript and carton artwork. The deviation list will contain “clone” type deviations that are unmapped copies in the artwork of mapped artwork text. By ignoring/marking these, the compare process can be sped up significantly. Be especially careful with unmatched text (orange marked text).

Tip 2:
If you are comparing other types of documents, they may be very different versions of the same document. You can use the Manual Mapping tool to map content deviations (yellow areas) as far as possible, but the main part of such documents has to be proofread manually.

23.4 The PDF document contains gray marked text

Gray markings indicate words that use illegal character codes (Its screen representation may still be correct). Usually, this involves text in symbol fonts (e.g. bullets) that have a custom encoding. If it occurs for only a few characters, your documents will still be comparable thanks to the map warning feature for illegal character codes (see 27.2 "Illegal Character Category" on page 1).

If large parts of the text are marked in gray, the comparison results are unreliable. You will have to ask the provider of your PDF document to submit properly encoded documents in future. Please see chapter “18.6 "Creating PDF Documents for Use with TVT" on page 1 for more details.
