40.21.1 Detecting Table Segments (Lines, Rows and Columns)

For MS Word/RTF documents, tables are automatically presorted by the logical cell information contained in the document. When using the Table Area Mode, TVT detects rows and columns by the spacing between the text segments within Table Areas. Lines are detected by their own spacing.

In a Table Area, the spaces between cells and columns are highlighted light green prior to the comparison and light gray afterwards. If the space between rows is not highlighted, the Table Area is in “Order by Columns” mode or the row factor needs to be adjusted. The space between columns should always be highlighted, but not be of interest if the Table Area is in “Order by line” mode. If columns are not properly detected, the column factor needs to be adjusted.

Line detection is only displayed in the “Order by line” mode by green or gray lines under the text line it is associated to. If there are no lines, the Table Area is not in “Order by line” mode or the Line Factor is not correctly adjusted.

By adjustment of the line, cell and column factor, the user can stipulate the minimum distance between them. A negative number inserted will allow overlap of lines/cells/columns, whereas a positive number will require a minimum physical space between them. Any table segment that does not have the minimum space will be combined with the adjoining table segment.

The formula for calculating table segment factors calculates as follows:

[Minimum Distance] = [Factor] x [Font Size]

E.g. a 0.3 column factor with 12 pt text results in a minimum distance of 3.6 pt between the columns.

40.21.1.1 Line Factor

Normally line factors are negative, as the word boxes for two adjacent lines tend to overlap a bit. If line factors were positive, adjacent lines would be detected as one. For European fonts, the preset line factor is -0.3. Fonts with higher word boxes will require higher negative values.

40.21.1.2 Row Factor

By definition, row factors must be larger than line factors. While the default value is 0.3, it might be necessary to reduce the value to 0.0 when the Table Area is inside a MS Word document.

40.21.1.3 Column Factor

Column factors are usually positive and should not allow word spacings to be identified as column breaks. As the typical space width is about ¼ of the font height, the minimal column factor should be at least 0.25. In order to be on the safe side, TVT has a preset column factor of 0.6.
