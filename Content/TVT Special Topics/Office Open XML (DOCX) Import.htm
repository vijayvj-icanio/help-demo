40.7 Office Open XML (.DOCX) Import

The DOCX format is Word’s default document format since Microsoft Office 2007. Unlike the .DOC format, it is documented and readable by other applications. The DOCX specification is very extensive and complex. Therefore, only important Word features are supported by the DOCX import. A distinction must also be made between feature imports for graphical representation and for text extraction. In general, text extraction has the higher priority; graphical representation is considered secondary. In cases where either the DOCX specification is unclear, or Word behaves differently, or there are differences between the different Word versions, the behavior of the newest Word version (Office 365) is preferred.

DOCX Import is done in the following successive steps:

  • Extracting the content of the DOCX container to a temporary directory
  • Loading the extracted files into the internal document object model (DOM)
  • Rendering the content from the internal DOM for display
  • Extracting the textual content from the DOM for comparison.

40.7.1 Graphical Representation

The same features are supported as enumerated in reference to 16.6.1 "Graphical Representation" on page 1, with the following exceptions:

  • Section breaks are supported.
  • Footnote/Endnote identifiers are identical to Microsoft Word.
  • The text/images from text boxes and drawing shapes are imported in the order in which they appear in the document.
  • Embedded images:
    • Borders are supported
    • Graphical effects applied to images in Word are not reproduced (color adjustment, grayscale, etc.)
  • External content is not loaded. Linked images are replaced with a default error marker.
  • Mathematic formulas/equations are not imported. They are replaced with a default error marker.
  • Vectorized content created in Word, like lines or shapes, are not loaded.

40.7.2 Textual Import

Content is transformed into Unicode. Whenever possible, TVT tries to represent symbol-encoded characters with their Unicode equivalent. This substitution is currently supported for some characters in the “Symbol” and “Wingdings” fonts included with Windows, and the “Zapf Dingbats” font.

When a font used in a Word file is not available on the current system, TVT will replace it with a fallback font. This font exchange only takes place in order to ensure a similar visual representation of the text. The text is then displayed in the fallback font, but the original font name and font attribute data is kept intact.

A warning message during file loading, can be activated upon request. The message can also be replicated on the report, if the 9.4.1 "Report Options" on page 1 in the User Settings are set accordingly.

40.7.3 Track Changes Feature

TVT shows changes the same way as Microsoft Word does in the “No Markup” display mode. Text is imported as if all changes have been accepted.

40.7.4 Macro-Enabled Document Import (*.DOCM)

When a document contains macros, Microsoft Word saves them in the Macro-Enabled document with the extension *.docm. Technically, they are DOCX files with macros, so they can be loaded as well. But the macros are neither loaded nor executed by TVT.

40.7.5 Strict Open XML Document Import (*.DOCX)

As of Microsoft Word 2013, it is possible to save documents in the Strict Open XML format. Files in this format have the same extension as normal Office Open XML files, which are also known as Transitional Open XML Documents. TVT can load both formats.

40.7.6 Other word processors

The Office Open XML format is an open format and other applications than Microsoft Word may save their documents in that format too. The DOCX import of TVT was tested with files saved from Microsoft Word and with manually created files. Thus, it can occur that TVT does not load DOCX files produced by other word processing applications.
