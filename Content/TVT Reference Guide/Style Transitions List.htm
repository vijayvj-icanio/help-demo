25.1.0.1 Style Transitions List

The Style Transitions list allows you to view all style differences between the documents. It is organized as a table, displaying the following information:

Checkbox:

By default, all rows are valid when opened for a first time in a project. Checking the checkbox will mark a transition as an unacceptable style deviation by highlighting the line in red as long as it is not selected.

This checkbox can have a third state besides checked and unchecked: When some instances of a style transition are overridden in the Style Transitions Details table (please see chapter 30.5.0.1 "Style Transition Details" on page 1), and the instances of the transition do not all have the same state, a question mark will be displayed in the checkbox instead.

Amount:

Number of instances of a particular transition.

 

Original Style

Description of the style in the original file(s).

 
Copy Style

Description of the style in the copy file(s).

 
Comment

For inputting a comment about the transition.

 

There is one mandatory button:

OK

Confirms that the Style Check has been performed

Additionally, the following buttons are displayed:

Reset This button resets the Style Check back to “Not performed”.
Automated Style Check…

Opens the Style Check and Style Conformity Mode window which allows you to select a Style Check mode and/or a Style Conformity criterion to be applied. For more information on this option, please see the See Special Topics "Style Check Mode Examples" on page 1 Style Check Mode Examples.
Check All

Puts check marks in all unchecked boxes and, therefore, marks all style transitions as unacceptable.

Uncheck All Removes the check marks in the boxes of the first column of the table and therefore marks all style transitions as acceptable.

To identify all instances of a particular style in the original or copy documents, select a line in the table. TVT will then show the transitions where the style matches the selected style in the original document by highlighting those instances in yellow in the “Original Style” column. Similarly, it will show the instances where the highlighted copy style is repeated by highlighting them in yellow in the “Copy Style” column.

It is possible to navigate from a particular mapped word in one document to the style transition in the Style Transitions list. Simply double right-click the word in the document and it will highlight that transition in gray in the Style Transitions list while showing the whole text section in the Details Area.

Comments regarding style transitions can be added either manually or automatically. To add a manual comment, select the style transition and then either press the “Enter” key or double-click on the “Comment” column. This opens the comment box for manual input of text through the keyboard.

+
Pastes the original style into the comment box.
+
Pastes the copy style into the comment box.

Saves the text and closes the comment box.

Important: This is the only way to save manually entered changes.
Closes the comment box without making changes to the comment.

You can also use the following shortcut keys to enter an automatic comment or to reset the Comment box:
Checks the checkbox. This enters an automatic comment into the Comment box in the form “Style is ... in original and ... in copy”.

or
Unchecks the checkbox. Resets the Comment box (clears any text).

After making a comment, the transition will be marked as unacceptable (in red) in the table and a check mark will be placed in the checkbox. Removing the comment will also reset the status and remove the check mark, marking the transition acceptable.
