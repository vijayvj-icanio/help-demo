5.1 Creating a New Project

When starting TVT, a window opens asking you to create a new project or to open an existing one. Click on “Create a New Project” to open the Project Setup window that will guide you to create your project:

 

1: Select the original document(s)

Select the original (reference) document(s) using the upper “Add” button, or in the case of a website URL, by copying the URL and using the "Paste URL" button. You can also 'drag & drop' files from your Windows Explorer or from one field to the other. Use Ctrl+Drag to copy and add a file from the Original to the Copy field and vice versa.

To remove an already added document, select it and use the “Remove” button.

If you prefer working with the keyboard, you can use the Shift key as you right-click the file, and then choose Copy As Path. Then choose the field where you want to add the document and use Ctrl+V. To delete an added file, press on the Delete key.

If you add more than one document, you can change their order in the list by highlighting the document and clicking the up and down arrow underneath the “Remove” button.

It is also possible to switch the position of a document from the original to the copy field and vice versa, by using the two arrow buttons . For multiple documents, the whole group can be switched to the original or copy field by using the double arrow button .

Please note: You can open the Project Setup window and add files to it, even when TVT is not opened yet. You do so by right-clicking on the files you want to compare in Windows Explorer, then selecting “Send to” and then selecting “Text Verification Tool” from the drop-down menu. TVT will open automatically and the files will be added to the Project Setup window. In this case, RTF or DOCX files will be added to the original field and PDF files to the copy field.

 

2: Select the copy document(s)

Select the copy (modified) document(s) to be compared with your original manuscript(s) using the lower “Add” button.

 

3: Name your project

To ensure that your project is easily identifiable and that it conforms to your company’s naming convention, TVT uses the name of your original document as the project name by default. In the case of URLs, which could be very long and also not have the name of the document easily identifiable, TVT will auto-populate the Project Name field with the current date and time. It is also possible to set the name of the copy document as the project name. After having selected the documents for comparison, click on the drop-down arrow next to the project name field and select the name of the copy document instead.

In addition, you can choose a different name for your project. Simply click in the “Project name” text field and type in the desired project name. It can include up to 100 characters.

The project name will be suggested as the file name, when saving the project.

 

4: Select your project setting

Your choice of project setting depends on the size of the documents to be compared and whether they contain repeated blocks of text or not. The project setting can be chosen directly from the Change drop-down menu. Alternatively, you can press the “Change” button and choose the setting from the Project Setting window. The Project Setting dialog contains a short description of the setting configuration.

TVT offers you several project settings optimized for the following types of documents:

The additional Project Settings for comparison of „Fachinformation“ and "Gebrauchsinformation" files can be acquired as an optional feature during your TVT installation.

Cartons

This configuration is designed for the comparison of packaging material such as cartons/folding boxes and labels against the original manuscript. It is optimized for one-page artwork documents, where single instances of text from the original are repeated several times in the copy (Clones).

For PDF documents, no line analysis takes place. Therefore, underline and strikethrough styles should be checked manually.

Cartons QRD Human

The Exclusion Word List “EMA QRD Labels” is applied to the original document.

For PDF documents, no line analysis takes place. Therefore, underline and strikethrough styles should be checked manually.

Cartons QRD Veterinary

The Exclusion Word List “EMA QRD Labels Veterinary” is applied to the original document.

For PDF documents, no line analysis takes place. Therefore, underline and strikethrough styles should be checked manually.

Cartons (without Clones)

This configuration is designed for the comparison of different versions of packaging material such as cartons/folding boxes and labels, where no repetition of text is expected in the copy.

For PDF documents, no line analysis takes place. Therefore, underline and strikethrough styles should be checked manually.

 

Documents

This configuration is designed for documents with repeated text blocks.

Documents (without hyphenation check)

The hyphenation check is disabled in this setting.

 

Leaflets

This is the ideal configuration for short documents with few repeated parts.

Leaflets (Complex Languages), Documents (Complex Languages)

These configurations allow easy handling of Thai, Arabic and Hebrew language content as well as further complex scripts.

Special settings for PDFs containing Arabic and Hebrew text: Rather than trusting the PDF extraction order, logical text order is computed from visual text order. The "Major Spacing" warning can be disabled.

Leaflets with special PDFs

This configuration is optimized for documents with few repeated parts, where the PDF files have been exported from a desktop publishing application without using a PDF printer driver. Try this setting instead of the default Leaflets setting if:

  • The PDF document shows differences in on-screen output and text extraction
  • Text highlighted in TVT is dislocated compared to the screen output of the PDF document
  • The PDF file has been created using the InDesign PDF Library
  • PDF-related bug tracking messages appear during document loading

 

The default project setting is “Leaflets”. However, you can change it or set another setting as default setting in the Project Setting window. Select a setting from the list of settings. Use the “Set Current Selection as Default” button at the bottom, if needed. Then confirm with “OK”.

Once you have selected the appropriate setting for your project, click on the “OK” button to complete the project creation.
