5.5.2 Reading Order Changes (ROC) and Order Breaks (OB)

A Reading Order Change indicates that there is a difference between the reading order of the original and copy files.

Please note: Reading Order Changes consist of Order Breaks. The grouping of Order Breaks to Reading Order Changes is optional and turned on by default. Here is an overview of the information displayed in the Project Window depending on whether the grouping is preselected or not:

The Reading Order Changes are displayed in the Deviation List as soon as the ROC button is toggled in the status bar. Selecting an entry has the following effects:

  • The text that has been moved gets selected on both sides.

  • The view of the Details Area changes to display the moved text, highlighted in green and framed by a thick red line, as well as the text before and after it in both original and copy files.

From this view, you can easily navigate to all text sections that are part of the selected Reading Order Change.

You can also view the Order Breaks that are part of a Reading Order Change individually. Expand the view by clicking on the plus sign at the beginning of a Reading Order Change entry. When a Reading Order Change is marked, all Order Breaks that are part of it are also marked.

Order Breaks can be recognized in the text of the compared documents by the Order Break Indicators. Like Reference Boxes (please see 41.3.4 Reference Boxes Button C:\TFS\Docs\FlareDoku\TVT Documentation\Output\Yuliana_Wong\TVT Web New Target\Content\TVT Reference Guide\Reference Boxes Button.htm file:///C:/TFS/Docs/FlareDoku/TVT Documentation/Output/Yuliana_Wong/TVT Web New Target/Content/TVT Reference Guide/Reference Boxes Button.htm Reference Boxes Button 1 1 on page 1), Order Break Indicators track any changes in the reading order of the text. However, while Reference Boxes are provided as supplementary information only, Order Breaks can be processed in the Deviation List (please see chapter 5.5.6 "Processing Deviations in the Deviation List" on page 1) and by using the Free Annotation and Error Area Modes. There are two different kinds of Order Break Indicators:

 

w This arrow marks the start of a continuously mapped text block, and indicates an Order Break Before
v This arrow marks the end of a continuously mapped text block, and indicates an Order Break After

 

Since the information they provide is very important, the Order Break Indicators are always visible.

When an Order Break is selected, red and blue arrows indicate the flow of text in both documents. For an Order Break After, the blue arrow indicates the following word in the original, while the red arrow indicates the following word in the copy. For an Order Break Before, the blue arrow comes from the previous word in the original, while the red one comes from the previous word in the copy. The Order Break arrows are shown only if their calculated origin and target are on the same page.

A Table Break is a special kind of Order Break. In this case, the order in a table is broken (please see chapter 5.2.2 "Defining Table Areas" on page 1). For detailed information describing a table break and how to handle it, please see See Special Topics "Working with the Table Area Mode" on page 1 Working with the Table Area Mode.
