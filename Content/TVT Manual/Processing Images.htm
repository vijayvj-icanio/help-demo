5.5.9 Processing Images

Statistically similar images are automatically linked into Image Pairs by TVT during the comparison process. If both the images in an Image Pair are identical, you can control how TVT handles these Image Pairs; in User Settings, under the Image Areas tab, in the Automatic Linking section, you can choose to "Link" them (this is the default behavior for Image Pairs, displaying the linked images in the Images List for processing), "Link & Accept" them, so they are displayed in the Images List and appear as automatically accepted, or automatically "Remove" them, so that they do not appear in the Images List at all.

You can also manually link images. After having drawn an Image Area around each image, right-click an Image Deletion or Insertion directly or in the Image List and select “Pick for Linking”. Then right-click the image you consider its counterpart and select “Link with Picked”. Another way is to press the CTRL key and then drag an entry in the Image List to its corresponding image. For those images that have not been automatically linked, you can also get suggestions from TVT for a corresponding image on the other document. In the context menu of an Image Area or an Image Insertion/Deletion in the Image List, select "Suggest Matching Image” and then one of the following alternatives: Best Match, 2nd Best Match and 3rd Best Match and TVT will navigate you to the respective match. When an Image Deletion and an Image Insertion are linked, their background color turns yellow the entry in the list is called “Image Pair”.

You can then inspect an Image Pair in the Image Comparison View that can be accessed through selecting "Image Comparison View" from the context menu of an Image Pair. You can also click on the "Image Comparison View" button in the Project Window toolbar instead. Images can be processed from the Image List in a way similar to the way Deviations are processed in the Deviation List. Please note that the automatic comment content for marked images vary from the automatic comment content for marked deviations. For instance, when an Image Deletion is marked, the automatic comment is “Image Missing in Copy”. All Information on missing images is then included in a summary comment located in the left upper corner of the exported version of the corresponding document. The summary comment contains a list of the missing images and their exact position in the document in which they appear.
