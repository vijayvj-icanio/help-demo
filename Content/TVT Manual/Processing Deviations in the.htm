5.5.6 Processing Deviations in the Deviation List

Each deviation is identified according to type, e.g.

  • Insertion: The copy contains text that is not present in the original
  • Deletion: Text contained in the original document is missing in the copy
  • Deviation: A difference is found between text in the original and the copy, e.g. “mg” in the original appears as “µg” in the copy
  • Warnings: Warnings are defined as mapped deviation pairs. They occur when non-identical words are mapped in the comparison process. Further details on Warnings can be found in the See Special Topics "Warnings" on page 1 Warnings.
  • Spacing Warnings: Spacing Warnings are issued if the spacing between characters in a string of letters has changed. There are two types of Spacing Warnings: Minor and Major, where Minor theoretically does not alter the meaning of the text and Major alters its meaning. Minor Spacing Warnings can be accepted in groups by unchecking the Minor Spacing Warning box in the Warning Settings window. (Further details on Spacing Warnings can be found in the See Special Topics "Spacing Warnings" on page 1 Spacing Warnings).
  • Clone (only available when the Carton setting is used and only in the copy document): This text is a duplicate of a mapped text sequence (master) located elsewhere in the copy document.
  • Correction(only available with imported annotations being text edit comments in the original): TVT checks whether the correction instructions in the original have been implemented in the copy. For more information on Corrections, please see 5.5.1.1 "Corrections" on page 1.

The page on which the deviation occurs is also identified and indicated.

Click on an entry in the Deviation List. This has the following effects:

  • The deviation is selected and centered in the original and the copy documents. You can press the Z key to zoom in on the deviation in both documents to the maximum zoom level allowing all the selected text to be displayed. By pressing the O key, the zoom level decreases to 100%.
  • The text or string of characters containing the deviation is displayed in the Compare Selections area of the Project Window. The text of the original document has a blue background; the text of the copy document has a red background. All characters with matching Unicode appear in blue, the non-matching ones in red.

If the entry is an Insertion or a Deletion, the following mark is shown in the document where the text has been inserted or deleted. By default this insertion mark is placed directly after the last word or punctuation mark that is mapped in both documents. The location of the symbol and the correction annotation can be adjusted by using the left and right cursor keys or the Alignment buttons   in the Project Window toolbar.

 

The user needs to decide:

  • If the selected deviation is an error that requires correction or if it can be accepted.
  • If there is an error that requires correction, and whether it appears in the copy, in the original, or in both documents.

Once you have determined the significance of a deviation, you can decide how it is to be handled. The toolbar at the bottom of the Project Window offers the following functions for processing deviations:

 

Accepts the current deviation in both documents

Marks the current deviation as an error in the copy document and automatically generates a comment on the action to be taken

Marks the current deviation as an error in the original document and automatically generates a comment on the action to be taken

Marks the current deviation as an error in both documents and automatically generates a comment on the action to be taken

These functions are also available in the context menu of the deviation, which is accessed by right-clicking on a deviation in the list.

As you work through the Deviation List, TVT automatically creates a comment on how the deviation is to be handled and then moves on to the next one in the list. Automatic comments can be edited. To do so, simply select the deviation and press the Enter key, or double click the comment field of the deviation, or click on the Edit Comment button located in the Project Window toolbar. You have to press Enter when you are finished editing the comment, in order to save the edited text. To discard the changes you made in the comment, press the Escape key. For further Deviation List actions, please see 6.2.2 "Project Window/Deviation List" on page 1.

In addition to selecting entries in the Deviation List, you can also double-right-click a deviation in the text of one of the documents. The corresponding entry will be then selected in the Deviation List.
