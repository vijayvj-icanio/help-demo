5.2.2 Defining Table Areas

In general, the layout of the text within tables should be checked manually. Some documents containing complex tables may require the use of the Table Area Mode to enable the mapping of text within tables. This also enables the detection of moved cells, called Table Breaks (please see chapter See Special Topics "Understanding Table Breaks" on page 1 Understanding Table Breaks).

To place a Table Area, click on the Table Area Mode button in the toolbar. Then click and drag to draw a green box over the text block. The size of the Table Area can be increased or decreased manually as needed by dragging on its edges.

A Table Area can be deleted by right-clicking on it and selecting “Delete” from the context menu. This function is only available when the Table Area Mode is active.

Once you have placed the Table Area, you can decide the order in which the table is to be read. Right-click on the Table Area and select from:

  • Order by Line
  • Order by Cell
  • Order by Column

You can also adjust the table properties. For more information, please see See Special Topics "Detecting Table Segments (Lines, Rows and Columns)" on page 1 Detecting Table Segments (Lines, Rows and Columns).

 

It is recommended to place Table Areas before comparing documents. However, you can apply Table Areas after the comparison. The option of applying Table Areas in the compared state is a convenience, in case you had forgotten to place them before comparison and did not notice until after making significant progress on your TVT project. Please note that applying Table Areas in the compared state has a number of caveats (please see See Special Topics "Applying Table Areas in the Compared State" on page 1 Applying Table Areas in the Compared State).
