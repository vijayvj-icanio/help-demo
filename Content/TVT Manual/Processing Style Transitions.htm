5.5.11 Processing Style Transitions including Style Conformity

Differences in style between the compared documents can be viewed by selecting the Style Check tab in the Project Window. The toggled table, referred to as the Style Transitions List, provides information on the style of all text that is present in both original and copy. Because of the need to tolerate document reformatting, it is possible to select the types of style transitions that are acceptable and those that are not.

In the Style Transitions List you can easily identify whether any font, font size or font attribute has been changed. All style transitions in the list are initially displayed as unchecked. Any transitions that are not correct should be checked/marked, either manually, or by applying the Automated Style Check explained below. The “Amount” column shows the number of mapped words that have the indicated style transition. The list is sorted by quantity. A large number of transitions usually indicates that the transitions are intentional changes. A small number of transitions may be a sign that the transitions are not intentional and may require a correction.

When a Style Transition is selected, TVT highlights all identical styles from the same document in yellow.

5.5.11.1 Automated Style Check

You can activate the Automated Style Check button at the bottom of the Style Check window. The Style Check and Style Conformity Mode window opens.

Style Check Options

You can choose one of the following options to perform the Automated Style Check:

  1. One-to-One Style Check

    The One-to-One Style Check marks all words that differ in style.

    It is most useful when comparing documents in the same format (e.g. Word vs. Word or PDF vs. PDF) in which all style attributes are intended to remain exactly the same.

     

  2. Simplified Style Check

    The Simplified Style Check compares only the following key style attributes on a one-to-one basis:

    • font weight (bold or regular)
    • slope (upright or italic/cursive)
    • underlining
    • strike-through
    • superscript or subscript.

    It is most useful for the comparison of documents that have expected font and size differences (e.g. Word vs. PDF). Style attribute information is also drawn from font names. If, for example, the word “bold” is already included as part of the font name, the font weight is considered to be bold.

     

  3. Adaptive Style Check

    The Adaptive Style Check is designed for comparing documents that have intended style differences (e.g. Word vs. PDF).

    It analyzes the style differences between the documents and makes assumptions about correct style transitions based on their frequency. The less frequent transitions are marked.

    The Adaptive Style Check is not recommended when the amount of text with incorrect style transitions is greater than the text with intended ones. If this is the case, you have to identify the incorrect style transitions manually.

     

  4. Simplified & Adaptive Style Check

    The Simplified & Adaptive Style Check combines the Simplified with the Adaptive Style Check. All style transitions spotted by at least one of them are marked.

    It is recommended for Word vs. PDF comparisons and is selected by default.

 

The 'None' option allows you to perform the Style Conformity check (please see below) only.

After selecting one of the Style Check options and confirming with OK, the number of marked style transitions is indicated in a pop-up message. These transitions are then highlighted in red and bear a check mark in the Style Transitions list.

 

Please note: Clones, being unmapped text, are not checked for style differences.

Please note: Independent of the way you have chosen to perform the Style Check, it will be considered as done by TVT only if you confirm it by clicking on OK at the bottom of the Style Transitions list.

The current status of the Style Check (performed or not) is shown in the Status bar . If you decide to redo the Style Check, click on the Reset button next to the OK button. The Style Check button is then restored to and the Style Check is considered as not performed yet.

 

Style Conformity

Additionally, you can activate the Style Conformity check. Style conformity can be important for regulatory compliance reasons (i.e. minimum font size for text as required by the regulatory authority) as well as for corporate branding (i.e. typography and font are often an integral part of the company brand and image). For instance, if guidelines require that “all text must be 8pt or greater”, or that “all text must be in font X or font Y”, text that does not conform to these guidelines will be highlighted as a conformity violation by TVT. The default style conformity check rules are: minimum font sizes 5, 6, 7, 8, 9, 10 and 11pt. To add additional font sizes and/or typefaces as required by your company, please contact your customer support partner.

 

For further information on checking styles or style conformity, please see See Special Topics "Style Check" on page 1 Style Check.
