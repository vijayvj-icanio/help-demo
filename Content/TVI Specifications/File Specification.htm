36 2. TVI File Specification

36.1 2.1. XML Header

The TVI format is based on XML. Therefore an XML header should be provided. TVT is able to read typical encodings.

 

<?xml version="1.0" encoding ="utf-8"?>

 

36.2 2.2. General File Structure

The TVI file is based on XML and provides TVT with information necessary to set up a project. It also contains target file names for the finalization of the project. In addition, it contains settings to overrule the settings in TVT. The overall tag structure is quite simple:

 

36.3 2.3. tvi element

The <tvi> element is the main node of a tvi file. It has only one <project> child element. For development purposes, the <tvi> element should be enriched by a reference to its schema, which adds basic syntax checking and intellisense in XML editors.

 

<tvi

    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

    xsi:noNamespaceSchemaLocation="Schema/tvi.xsd"

    >

36.4 2.4. project element

The <project> element must contain at least two <file> elements. There can only be one <project> element in a tvi file.

 

<project mode="init" settings="options1" name=”my project”>

36.4.1 2.4.1. mode attribute

The possible values for the mode are “init”, “in” and “inout”, which represent the operation modes described in 1.1. Operation Modes above.

36.4.2 2.4.2. settings attribute

The value of the settings attribute specifies a project settings file that comes with the installed TVT version. The name must be specified without any extension or path. For the complete list of possible values, refer to the “projectsettings” folder of the TVT installation path. Typically, the following values are used:

options1    Leaflets settings

options2    Cartons settings

options3    Documents settings

36.4.3 2.4.3. name attribute

The optional name attribute sets the TVT project name. If omitted, TVT will provide a suitable default. The project name may not be empty, and may not contain characters that are not valid in file names. TVT does not permit changing the project name in the “in” and “inout” modes.

36.4.4 2.4.4. id attribute

The id attribute is currently not used by TVT.

36.4.5 2.4.5. auto_compare attribute

This is an optional attribute. If this attribute has the value “true” or “1”, TVT will automatically initiate the comparison (as if the Compare button were clicked) as soon as it has successfully loaded all files. Setting the attribute to other values, or not setting it at all, has no effect.

36.4.6 2.4.6. style_check attribute

This is an optional attribute. The following four values can be used to run an automatic style check: “OneOnOne”, “Adaptive”, “SimplifiedAdaptive”, “Simplified”. Styles can only be checked in compared projects. Therefore, the TVI must either specify a compared project to be loaded or create a new project with an automatic comparison (2.4.5. auto_compare attribute) enabled. If the loaded project already contains style check results, they are reset.

36.4.7 2.4.7. style_conformity_check attribute

This is an optional attribute. The value of the settings attribute specifies a style conformity file that comes with the installed TVT version. The name must be specified without any extension or path. For the complete list of possible values, refer to the “Additional Input” folder of the TVT installation path. For example, “fontsize_min_5pt” is a valid case. The requirements for using style conformity check are the same as for style check (2.4.6. style_check attribute).

36.4.8 2.4.8. auto_finalize attribute

This optional attribute may only be used in the “inout” mode. Setting this attribute to “true” or “1” has the following effect: After a successful comparison (i.e. if the comparison is not interrupted by pressing the Cancel button), TVT will immediately begin the project finalization and then close TVT. The finalization omits the finalization dialog (please see 1.1.3. InOut Mode above) and is automatically set to “Finish Project”.

The auto_finalize attribute can be used in combination with the auto_compare attribute to run TVT in a semi-unattended manner. If both attributes are set to “true”, TVT will automatically load the input files, compare them, finalize the project by writing all specified output files, and then close. (Note that this is not a true unattended mode, since TVT might display warning dialogs that need to be dismissed by a user.)

36.4.9 2.4.9. mark_unmarked attribute

This optional attribute may only be used in the “inout” mode, and either the auto_compare attribute is set to “true”, or an already compared project is loaded. If set to one of the three permitted values, the finalization that is initiated by selecting “Finish Project” will do the following: Before writing the ouput files, all Deviations that have not already been marked by a user will be marked in the specified manner. Deviations that have been marked by a user will be left untouched. The permitted values for this attribute are:

original   marks all unmarked Deviations as if “Mark in Original” had been clicked

copy        marks all unmarked Deviations as if “Mark in Copy” had been clicked

both        marks all unmarked Deviations as if “Mark in Both” had been clicked

The “Cartons” settings (please see 2.4.2. settings attribute above) should not be used in conjunction with the “original” or “both” values; only “copy” is permitted in this combination.

36.4.10 2.4.10. username attribute

This optional attribute can be used to set the user name to which all actions (e.g. comparing/finalizing a project, marking deviations etc.) will be attributed. The user name will have “(TVI)” appended in such logs to clarify the source of the user name.

36.5.1.1 2.5.1.1. document

The document type represents a file that is going to be loaded into TVT for comparison. Multiple file elements with identical sides can be chained, in order to load multiple files either as manuscript or copy. The document type serves as fallback if no type attribute is specified in a file element.

In the “inout” mode, the “path_out” attribute may be used to specify the path to the directory, in which the exported PDF document will be written. If this attribute is omitted, TVT will not create an exported version of this document. It is therefore valid to export only a subset of the documents in the project.

36.5.1.2 2.5.1.2. project

When a project is given through the path_in attribute, TVT will automatically load it. Although saved projects contain local links to their document files, these must be additionally specified in the TVI file. This way, it is ensured that the documents may be loaded automatically, even if their names have been changed in the meantime (e.g. by the DMS they originate from).

36.5.1.3 2.5.1.3. ignoretemplate

Exclusion templates can be applied by using the ignoretemplate attribute. The document side, on which the template will be used, has to be specified and it will affect all documents there.

36.5.1.4 2.5.1.4. excludelist

Like the exclusion template, an exclusion list may be applied to either document side by using the excludelist attribute.

Use “none” as value to disable the default Exclusion Word List without specifying a list.

36.5.1.5 2.5.1.5. criticalwords

Critical Word Lists are applied to both document sides and only one may be named. A full path to a custom Critical Word List can be specified.

36.5.1.6 2.5.1.6. defaultcriticalwordlists

The defaultcriticalwordlist attribute can be used to apply any of the existing Critical Word Lists. The names of the lists should be specified without a path, include an extension and be separated by the Vertical Line character ‘|’ (U+007C). For instance, entering “si_units.tvcrit|trademarks.tvcrit” will activate two Critical Word Lists. For a complete list of possible values, refer to the “Additional Input” folder in the TVT program directory.
Use “none” as value to disable the default Critical Word List without specifying a list.

36.5.1.7 2.5.1.7. report

By setting the report attribute, TVT can be told to generate a project report in the file format of the extension. Report settings are applied as provided by the TVT User Settings.

36.5.1.8 2.5.1.8. result

The result file is a simple text document, which is generated by TVT when it is closed in “inout” mode. The finalization action set in TVT will be propagated to this file. It may contain one of the following texts:

  • FINISHED

  • SAVED

  • CANCELLED

It is encoded in UTF-16LE. Before evaluating all TVT result files, the TVT Wrapper has to wait until the “result” file is properly created, filled out and closed. If the result file specification is omitted, the file will not be generated.

36.5.2 2.5.2. side attribute

For files that are relating to a certain side of the project, it is necessary to specify either “original” or “copy” as value of the side attribute.

36.5.3 2.5.3. path_in attribute

All input files require the “path_in” attribute. It may contain a full path name or a relative path as seen from the location of the TVI file.

36.5.4 2.5.4. path_out attribute

Output files must be set in the “path_out” attribute. Some file types (e.g. documents and projects) may combine “path_in” and “path_out” attributes. Except for project files, documents should not be overwritten by TVT.

In addition to PDF, exported files can be in the XFDF file format as well.

 

36.6 2.6. settings element

The settings element can be used to set permissions individually, in addition to the mode. The structure contains categories with several properties beneath, e.g.:

<settings>

<CATEGORY ID="PERMISSIONS">

<PROPERTY ID="PROVIDE_EXPORT_FILE_ORIG" VALUE="1"/>

</CATEGORY>

</settings>

36.6.1 2.6.1. category element

The category is used to group the permissions. In TVI only one category is used and it is named "PERMISSIONS".

36.6.2 2.6.2. property element

In the category element PERMISSIONS, multiple permissions can be defined. Dependent on the mode, some functions are not enabled in TVT or not executed automatically in this particular mode.
