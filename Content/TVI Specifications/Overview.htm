27 1. Overview

 

Interfacing TVT with the TVT Wrapper is accomplished by a simple file-based interface. The following tasks are required for the initialization:

  1. Creation of a temporary folder for the project files on the computer on which TVT is installed.

  2. Provision of all files necessary for the comparison in this folder.

  3. Creation of a Text Verification Initializer (TVI) file which is also placed in the temporary folder.

  4. Running TVT.exe with the path to the TVI file as command line parameter.

As a result, a project, which is preconfigured with the documents and settings specified in the TVI file, is created. During TVI file loading, TVT will check the settings whether the settings are consistent and present helpful messages if not.

 

In addition to the “TVI” extension, TVT 12 registers the “TVI12” extension. If a previous version of TVT is installed, it is possible to open existing TVI files with the existing TVT version and open only the new TVI12 files with TVT 12.

 

In “inout” mode (please see 1.1.3. InOut Mode below), output files are fixed and will be interpreted by the TVT Wrapper application:

  1. Wait for the generation of the result file by TVT.

  2. Process the content of the result file and act accordingly:

    1. Cancel the project.

    2. Save the project in the Data Management System (DMS) for further processing.

    3. Finish the project and save the result files to the DMS.

27.1 1.1. Operation Modes

27.1.1 1.1.1. Init Mode

The initialization will only start TVT and create a project to compare the provided files. If the initializer specifies additional input, e.g. critical word lists or exclusion word lists, those can be overridden by the user. All output generation must be manually triggered and can be saved to freely chosen locations.

27.1.2 1.1.2. In Mode

The only difference to the “init” mode is that all input files are fixed, and cannot be changed in TVT. All output generation must be manually triggered and can be saved to freely chosen locations.

27.1.3 1.1.3. InOut Mode

All input and output files are fixed, and cannot be changed in TVT. Output files will be generated as specified in the TVI file. Output files that have not been specified, will be omitted.

In the TVT user interface, this mode disables all manual export and save options. When closing TVT, a dialog box asking for a finalization action appears:

 

 

There are three options which can be communicated to the TVT Wrapper in the result file:

    • Finish Project

    • Save Project

    • Cancel Project
